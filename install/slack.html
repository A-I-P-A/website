Install AIPA for Slack

Choose the right mode for how you want AIPA to work in your workspace.

AIPA Bot Recommended

AIPA joins your channels as a bot. It can read messages and send replies labeled [via AIPA]. Other users see the bot as a separate user. Workspace admins can easily manage permissions.

Bot scopes: channels:history, channels:read, chat:write, groups:history, groups:read, im:history, im:read, mpim:history, mpim:read, users:read

Install AIPA Bot

AIPA Personal Advanced

AIPA reads and writes as you. No bot user, no [via AIPA] prefix. Messages appear as if you typed them. Other users cannot tell the difference. Use this for the full digital twin experience.

User scopes: channels:history, channels:read, chat:write, groups:history, groups:read, im:history, im:read, mpim:history, mpim:read, users:read

By installing AIPA Personal, you authorize AIPA to send messages as your Slack identity. You are responsible for all messages sent. Terms of Service apply.

Install AIPA Personal

What happens after install

  • AIPA connects to your workspace and discovers channels
  • Open the AIPA iOS app, your workspace appears automatically
  • Create an agent and link it to the channels you want to monitor
  • For AIPA Bot: invite the bot to channels with /invite @AIPA
  • For AIPA Personal: no invite needed, AIPA sees everything you can see

Which should I choose?

AIPA Bot AIPA Personal
Messages appear as Bot user (labeled) You (indistinguishable)
Channel access Must be invited to each channel Sees everything you can see
Admin approval Standard bot install User-level OAuth only
Auto-reply capable Yes (as bot) Yes (as you)
Best for Getting started, team visibility Digital twin, full autonomy

Privacy and Security

All messages are processed on your device. AIPA does not store your Slack data on any server. See our Privacy Policy for full details.
